# 世说蓝语项目备份指南

## GitHub备份操作步骤

### 1. 日常备份（提交新更改）

```bash
# 检查当前状态
git status

# 添加所有更改
git add .

# 提交更改（请提供有意义的提交信息）
git commit -m "描述你的更改内容"

# 推送到GitHub
git push
```

### 2. 查看提交历史

```bash
# 查看提交历史
git log --oneline

# 查看详细提交历史
git log
```

### 3. 恢复到之前的版本

```bash
# 查看提交历史，获取提交ID
git log --oneline

# 恢复到指定提交（这将创建一个新的提交）
git revert <commit-id>

# 或者强制回退到指定提交（谨慎使用）
git reset --hard <commit-id>
git push --force-with-lease
```

### 4. 创建新分支进行实验性更改

```bash
# 创建并切换到新分支
git checkout -b feature-branch-name

# 在新分支上进行更改...

# 切换回主分支
git checkout main

# 合并新分支
git merge feature-branch-name

# 删除已合并的分支
git branch -d feature-branch-name
```

### 5. 从其他设备同步项目

```bash
# 克隆项目到新设备
git clone https://github.com/lsh3300/shishualanyu-app.git

# 进入项目目录
cd shishualanyu-app

# 拉取最新更改
git pull
```

## 重要文件备份

### 环境变量文件
- `.env.local` 文件包含敏感信息，不应提交到Git
- 请手动备份此文件到安全位置

### 数据库备份
如果项目使用数据库，请定期备份数据库：
1. 导出数据库结构和数据
2. 将备份文件存储在安全位置
3. 记录备份日期和版本

## 备份策略建议

1. **每日备份**：完成重要功能开发后提交到GitHub
2. **版本标记**：完成重要里程碑时创建Git标签
   ```bash
   git tag -a v1.0.0 -m "版本1.0.0发布"
   git push origin v1.0.0
   ```
3. **定期检查**：每月检查备份完整性
4. **多地点备份**：除了GitHub，考虑在其他云存储服务上也保留副本
