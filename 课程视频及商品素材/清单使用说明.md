# 课程作品清单使用说明

## 📋 清单文件
**文件位置**: `课程作品清单.json`

## 📊 统计概览
- **作者数量**: 32人
- **课程作品**: 58个
- **产品图片**: 30个

## 📁 清单结构

### 1️⃣ 课程作品 (courseWorks)
每个作品包含以下信息：
```json
{
  "author": "作者姓名",
  "workName": "作品名称",
  "videoPath": "视频文件完整路径",
  "effectImagePath": "效果图完整路径（.png）"
}
```

**示例**:
- 作者: 姚依然
- 作品: 蝴蝶纹样
- 视频: `202136600128-姚依然-蝴蝶纹样（有声）.mp4`
- 效果图: `蝴蝶纹样.png`

### 2️⃣ 产品图片 (productImages)
每个产品包含以下信息：
```json
{
  "productName": "产品名称",
  "imagePath": "产品图完整路径（.jpg）",
  "relatedAuthor": "相关作者（仅供参考，商品页面可不显示）"
}
```

**示例**:
- 产品名: 吉祥如意锦鲤图1
- 图片路径: `吉祥如意锦鲤图1.jpg`
- 相关作者: 姚依然（商品页面可不显示）

## 🎯 使用场景

### App课程页面展示
使用 `courseWorks` 数据：
- 展示作者姓名
- 展示作品名称
- 播放教学视频
- 展示效果图

### App商品页面展示
使用 `productImages` 数据：
- 展示产品图片
- 产品名称
- **不需要**显示作者信息（`relatedAuthor`字段仅供记录）

## 🔍 注意事项

1. **效果图对应规则**: 
   - 与视频文件名称对应的.png文件是视频最终制作效果图
   - 例如: `蝴蝶纹样（有声）.mp4` 对应 `蝴蝶纹样.png`

2. **产品图识别规则**:
   - 其他名称的.jpg图片（如"吉祥如意锦鲤图1.jpg"）是产品图
   - 用于商品页面售卖及展示

3. **路径格式**:
   - 所有路径使用正斜杠 `/` 格式
   - 路径为绝对路径

4. **部分作品可能缺少效果图**:
   - 当 `effectImagePath` 为 `null` 时表示未找到对应效果图

## 📝 原始文件结构保持不变
所有原始文件保持在原位置，清单仅作为索引和记录使用。
